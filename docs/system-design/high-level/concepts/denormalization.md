## Theory

Adding redundant data to optimize read performance.

**Techniques:**
- Storing computed values
- Duplicating data
- Flattening hierarchies

**Trade-offs:**
- Faster reads
- Data redundancy
- Complex updates
- Data inconsistency risk

**When to Use:**
- Read-heavy workloads
- Performance critical
- After measuring bottlenecks
